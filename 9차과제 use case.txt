1. slide 반응형으로 작성  
  1-1. prev 버튼 클릭시 해당 방향대로 슬라이드가 계속 무한 루프
        슬라이드 동작은 한 섹션이 지나가는게 보이는 효과
        prev는 왼쪽에서 오른쪽으로 지나감
  1-2. next 버튼 클릭시 해당 방향대로 슬라이드가 계속 무한 루프
        슬라이드 동작은 한 섹션이 지나가는게 보이는 효과
        next는 오른쪽에서 왼쪽으로 지나감
  1-3. next 방향으로 슬라이드가 무한루프 된다. (처음부터 돌고 있음 setInterval)
  1-4. stop 버튼을 클릭시 돌아가고 있는 슬라이드가 멈추고, play 버튼으로 바뀐다. 해당 텍스트도 바뀐다.
  1-5. play 버튼을 클릭시 멈춰있는 슬라이드가 다시 next 방향으로 돌아가고 stop 버튼으로 바뀐다. 해당 텍스트도 바뀐다.
  1-6. paging 버튼이 있고, 클릭시, 해당 슬라이드로 간다.

     2-1-1. hover시, 메뉴가 선택된 효과가 따라다니는 효과로 지정 (시간 없으면 패스)
  2-2. 상위 메뉴에 hover하고 하위 메뉴에 hover 하고 있는 동안에는 하위 메뉴는 계속 보여진다.
  2-3. 2-2 상태에서 하위 메뉴 밖으로 마우스를 이동하면 하위 메뉴가 안보인다.
  2-4. 2-1 상태에서 메뉴 밖으로 마우스를 이동하면 하위 메뉴가 안 보이고,
        해당 상위 메뉴가 선택된 효과도 사라진다.

**접근성**
1. 처음 탭키를 누르면 '본문 바로가기' 라는 버튼이 보이면서 엔터키 사용시
슬라이드 prev 버튼으로 가게 된다.
'본문 바로가기' 에서 엔터키를 치지 않고 탭키를 누르면 탭키가 로고부터 시작한다.
2. 탭키는 마크업에 나열되어 있는 순서대로 이동하는데 button 이나 a 링크를 따라 이동한다.
3. 메뉴의 경우 상위 메뉴 아래 하위 메뉴를 다 훑고 다음 상위 메뉴를 훑는다.
4. 슬라이드의 경우 prev ->현재 보이는 슬라이드 내 button이나 a 링크 -> next -> paging 버튼 순으로 타게된다.
여기서 현재 보이는 슬라이드 라는 것은 슬라이드가 무한루프로 돌고 있기 때문이다.
5. 팝업의 경우 해당 블록에서 엔터키를 누르면 팝업이 뜨고
이때 팝업내 button 이나 a 태그가 없으면 바로 close 버튼으로 focus가 가 있어야 한다.
이때 이 포커스 상태에서 탭키를 눌러도 이동하지도 않고, 팝업이 닫히지도 않는다.
팝업을 닫으려면 엔터키를 눌러주면 된다. 팝업이 닫히면 해당 팝업이 열렸던 블록에 focus가 남아있으면 된다.
6. 탭키의 포커스가 가장 마지막에 머물러 있을 때, 다시 탭키를 클릭시 다시 처음부터 시작한다.
